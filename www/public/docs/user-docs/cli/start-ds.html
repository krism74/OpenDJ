The start-ds Utility

The start-ds utility may be used to start the Directory Server, as well as to provide general information about the product.

It may be run without any arguments, in which case it will start the server as a background process. In this case, the script will not exit until the server has either started successfully or has encountered an error that prevents it from starting.

The arguments available for this utility include:
  • -N
    --nodetach

    This argument will cause the Directory Server to start as a foreground process which does not detach from the terminal. When the server is running in this mode, it may be stopped using the stop-ds tool from another window, or by pressing Ctrl+C in the terminal in which the server is running.

  • -V
    --version

    This argument will cause the Directory Server to display its version information and exit rather than attempting to start the server.

  • -F
    --fullVersion

    This argument will cause the Directory Server to display a more verbose set of version information than will be displayed when using the --version option. It will also exit after displaying the version information rather than actually attempting to start the server.

  • -s
    --systemInfo

    This argument will cause the Directory Server to display a set of general information about the system on which it is installed. It will then exit rather than attempting to actually start the server.

  • -H
    --help

    This argument will cause the Directory Server to display usage information and then exit rather than actually attempting to start the server.
