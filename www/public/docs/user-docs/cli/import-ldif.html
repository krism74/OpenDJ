The import-ldif Utility

The import-ldif utility may be used to populate a Directory Server backend with data read from an LDIF file, or with data generated based on a MakeLDIF template. In most cases, it is significantly faster than adding entries using LDAP.

In order to use this tool, the --backendID argument is required to specify the backend into which the data should be imported, as well as either the --ldifFile or templateFile option to specify the data source. The complete set of options that may be used with this tool is:
  • -n {backendID}
    --backendID {backendID}

    This argument specifies the backend ID of the backend into which the data should be imported. The available backends in the server may be determined using the list-backends tool.

  • -l {path}
    --ldifFile {path}

    This argument specifies the path to the LDIF file containing the data to import into the specified backend. This argument must not be used in conjunction with the --templateFile argument.

  • -t {path}
    --templateFile {path}

    This argument specifies the path to a MakeLDIF template file that should be used to generate the data to import into the backend. If this argument is provided, then it is recommended that the --randomSeed argument also be used to ensure that the data generated for a given template is consistently the same.

    Note that if the path provided for this argument is an absolute path, then it will be interpreted as-is. If it is a relative path, then it will first be interpreted as relative to the config/MakeLDIF directory below the server installation. If no such template file exists relative to that path, then the server will look for the file relative to the user's current working directory when the tool was invoked.

  • -a
    --append

    This argument indicates that the data should be appended to the data that already exists in the backend, rather than clearing all data in the backend before starting the import.

  • -r
    --replaceExisting

    This argument indicates that if an entry to import already exists in the target backend, it will be replaced with the content from the import. If this argument is not provided, then existing entries will not be overwritten. This is only applicable if the --append option has also been provided.

  • -b {baseDN}
    --includeBranch {baseDN}

    This argument specifies the base DN for a branch of the data that should be included in the import. This argument may be used multiple times to specify multiple include base DNs. If this argument is provided, then any entries contained in the import source that are not at or below one of the provided include base DNs will be skipped.

  • -B {baseDN}
    --excludeBranch {baseDN}

    This argument specifies the base DN for a branch of the data that should be excluded from the import. This argument may be used multiple times to specify multiple exclude base DNs. If this argumnt is provided, then any entries contained in the import source that are at or below one of the provided exclude base DNs will be skipped.

    Note that the use of the --excludeBranch option takes precedence over the --includeBranch option, so if an entry is at or below a DN contained in both the include and exclude lists, it will not be included. This capability makes it possible to include data for only part of a branch (e.g., all entries below "dc=example,dc=com" except those below "ou=People,dc=example,dc=com").

  • -i {attribute}
    --includeAttribute {attribute}

    This argument specifies the name of an attribute that should be included in the import. This argument may be used multiple times to specify multiple include attributes. If this argument is provided, then any attributes not listed in this set will be omitted from the entries that are imported.

  • -e {attribute}
    --excludeAttribute {attribute}

    This argument specifies the name of an attribute that should be excluded from the import. This argument may be used multiple times to specify multiple exclude attributes. If this argument is provided, then any attributes listed in this set will be omitted from the entries that are imported.

  • -I {filter}
    --includeFilter {filter}

    This argument specifies a search filter that may be used to identify entries that should be included in the import. This argument may be used multiple times to specify multiple include filters. If this argument is provided, then any entry in the import source that does not match an include filter will be skipped.

  • -E {filter}
    --excludeFilter {filter}

    This argument specifies a search filter that may be used to identify entries that should be excluded from the import. This argument may be used multiple times to specify multiple exclude filters. If this argument is provided, then any entry in the import source that matches an exclude filter will be skipped.

    Note that the use of the --excludeFilter argument takes precedence over the --includeFilter option, so if an entry matches filters in the include and exclude lists, then the entry will be skipped.

  • -R {path}
    --rejectFile {path}

    This argument specifies the path to a file that will be used to hold any entries that should be included in the import (i.e., not excluded by the include/exclude base DN or filter lists) but are rejected by the server for some reason. A comment will be included before the entry indicating the reason that it was rejected. If this argument is not provided, then no reject file will be written.

  • -O
    --overwriteRejects

    This argument indicates that the server should overwrite the specified reject file if it already exists. If this option is not provided, then any rejected entries will be appended to the file rather than overwriting it. This option is only applicable if the --rejectFile argument is also provided.

  • -S {seed}
    --randomSeed {seed}

    This argument specifies the seed that should be used for the random number generator when generating entries from a MakeLDIF template. Seeding the random number generator with a particular value may be used to ensure that the same template and random seed will always generate exactly the same data.

  • -s
    --skipSchemaValidation

    This argument indicates that the server should not perform any schema validation on the entries as they are imported. This may provide improved import performance, but it should only be used if it is known that import data is valid.

  • -c
    --isCompressed

    This argument indicates that the file containing the LDIF import data is compressed. It should be compressed using the GZIP format, which is the format used by the --compressLDIF option of the export-ldif tool.

  • -y
    --isEncrypted

    This argument indicates that the file containing the LDIF import data is encrypted. Only data encrypted with the --encryptLDIF option of the export-ldif tool.

  • -q
    --quiet

    This argument indicates that the import should be performed in quiet mode, in which no output will be generated unless a significant error occurs during the import process.

  • -H
    --help

    This argument will display usage information for the import-ldif utility and will exit without attempting to import any data.
